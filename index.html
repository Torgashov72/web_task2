Лабораторная работа №2

Курс: Сетевые технологии | HTTP-протокол

Задание 0: Подготовка файлов

Файлы (index.php, file.tar.gz, image.png) загружены в репозиторий GitHub, после чего размещены на учебном веб-сервере в каталоге /www/web_task2.

Доступность файлов проверена в браузере по адресу: http://u82364.kubsu-dev.ru/web_task2/

Задание 1: GET главная страница (HTTP/1.0)

Выполнить GET-запрос к главной странице сервера с использованием протокола HTTP 1.0. Этот протокол не поддерживает постоянные соединения — после каждого запроса соединение закрывается. 

GET /web_task2/ HTTP/1.0
Host: u82364.kubsu-dev.ru

Задание 2: GET index.php (HTTP/1.1)

Получение страницы index.php методом GET (протокол HTTP/1.1). В отличие от HTTP 1.0, этот протокол поддерживает постоянные соединения. 

GET /web_task2/files/index.php HTTP/1.1
Host: u82364.kubsu-dev.ru
Connection: close

Задание 3: Размер файла file.tar.gz (HEAD)

Определение размера файла file.tar.gz без скачивания (метод HEAD). HEAD запрашивает только заголовки ответа без тела, что позволяет получить метаинформацию о файле без его загрузки. 

HEAD /web_task2/files/file.tar.gz HTTP/1.1
Host: u82364.kubsu-dev.ru
Connection: close

Задание 4: Медиатип image.png (HEAD)

Определение типа содержимого (медиатипа) файла image.png (метод HEAD). Медиатип (MIME-type) определяется из заголовка Content-Type. 

HEAD /web_task2/files/image.png HTTP/1.1
Host: u82364.kubsu-dev.ru
Connection: close

Задание 5: Отправка комментария (POST) — 2 балла!

Отправка комментария на сервер по адресу /index.php (метод POST). POST передает данные в теле запроса, что требует указания заголовков Content-Type и Content-Length. 

POST /web_task2/files/index.php HTTP/1.1
Host: u82364.kubsu-dev.ru
Content-Type: application/x-www-form-urlencoded
Content-Length: 31
Connection: close

comment=Test+comment+from+student

Задание 6: Первые 100 байт (Range)

Получение первых 100 байт файла file.tar.gz (заголовок Range). Это позволяет получить только указанный диапазон байт, не скачивая весь файл. 

GET /web_task2/files/file.tar.gz HTTP/1.1
Host: u82364.kubsu-dev.ru
Range: bytes=0-99
Connection: close

Задание 7: Кодировка index.php (HEAD)

Определение кодировки страницы index.php (метод HEAD). Кодировка указывается в заголовке Content-Type. 

HEAD /web_task2/files/index.php HTTP/1.1
Host: u82364.kubsu-dev.ru
Connection: close

Использование Git в работе

В ходе выполнения задания использовалась система контроля версий Git для управления файлами и синхронизации с сервером. Основные команды:

Команда git init

git init

Инициализация локального репозитория в текущей директории.

Команда git add

git add .

Добавление всех файлов в индекс (стадию подготовки коммита).

Команда git commit

git commit -m "Задание 2: HTTP-запросы + скриншоты"

Создание коммита с описанием изменений.

Команда git clone

git clone https://github.com/ваш_логин/web_task2.git

Клонирование репозитория с GitHub на локальный компьютер.

Команда git pull

git pull origin master

Обновление локальной копии репозитория с сервера.

Команда git push

git push origin master

Отправка изменений с локального компьютера на удалённый репозиторий.

Команда git remote add

git remote add origin https://github.com/ваш_логин/web_task2.git

Настройка соединения с удалённым репозиторием.

Команда git status

git status

Проверка состояния репозитория (изменённые, новые, удалённые файлы).
